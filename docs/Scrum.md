Scrum на уровне эпиков

Ниже эпики в логике Spec-First + Gate-Driven, но на уровне эпиков (без расписывания сторис на каждый файл; это сделаем позже, когда перейдём к деталям).

Epic 1 — Repository & Solution Bootstrap
Цель: создать каркас, чтобы дальнейшая разработка не превращалась в “копипасту по папкам”.
Выходные артефакты:
Git репозиторий с .sln, проектами src/*, tests/*, examples/*, docs/*
общие сборочные настройки (nullable, warnings as errors, форматирование)
CI pipeline: сборка + тесты на push/PR

Epic 2 — Core Contract
Цель: зафиксировать “контракт” минимального ядра, чтобы не ломать API на каждом шаге и чтобы Geometry/Kalman строились поверх стабильного фундамента.
Содержательно это про:
финальное имя типа (например UDouble) и финальные имена полей/свойств (Mean, Variance, StdDev, допустимые алиасы)
набор фабрик (включая фабрики, учитывающие ошибку округления float/double)
политика про неявные преобразования, сравнения, NaN/Infinity/исключения
Выходные артефакты:
docs/core-model.md (контракт + допущения: нормальность, независимость, линейная пропагация)
замороженные публичные сигнатуры Core

Epic 3 — Core Implementation
Цель: реализовать Core строго по контракту, без “протечек” геометрии и Калмана.
Выходные артефакты:
реализованный Uncertainty.Core (тип, операторы, базовые функции, базовая статистика)
корректные фабрики FromDoubleWithRounding / FromFloatWithRounding (как ты сформулировал)
понятные исключения/поведение на краях (0, отрицательное под Sqrt, домены для Acos/Asin/Ln и т.п.)

Epic 4 — Core Test Baseline
Цель: превратить Core в стабильную основу, которую не страшно публиковать и наращивать.
Выходные артефакты:
unit-тесты на формулы дисперсий для + - * /
тесты на фабрики округления (ULP, субнормалы, ноль, большие числа)
тесты на функции (через производные/ожидаемые формулы дисперсии), плюс регрессионные кейсы “малые значения” если ты сохраняешь специальные ветки

Epic 5 — Geometry Layer (Angles/Rotations)
Цель: вынести всё, что связано с периодичностью и вращениями, в отдельный пакет, не трогая контракт Core.
Выходные артефакты:
Uncertainty.Geometry с минимальным набором типов и операций, которые реально нужны твоим текущим сценариям
тесты на нормализацию углов, работу около границ диапазона, композицию вращений
1 пример в examples/ (например: преобразование измерений угла/курса и оценка разброса)

Epic 6 — Kalman Layer
Цель: упаковать существующий фильтр Калмана в библиотечный, переиспользуемый вид.
Выходные артефакты:
Uncertainty.Kalman (линейный Калман как минимум)
тесты на “эталонные” сценарии (простая 1D/2D модель, проверка снижения ковариации, консистентность размерностей)
пример в examples/ (сенсор-фьюжн на игрушечной модели)

Epic 7 — Packaging & Release (NuGet)
Цель: сделать из репозитория продукт, который ставится одной командой и ведёт себя предсказуемо.
Выходные артефакты:
корректные .csproj метаданные NuGet (id, version, authors, repository, license, tags)
Release pipeline (сборка пакетов, подпись/символы — по желанию)
публикация Uncertainty.Core (и опционально Geometry/Kalman, если готовы)
